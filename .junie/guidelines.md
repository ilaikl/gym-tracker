# Task Execution Rules

- Only execute tasks explicitly requested.
- Mark tasks [x] upon completion.
- Do not modify `requirements.md` without explicit instruction.
- If scope changes:
    - Update `requirements.md`
    - Update `plan.md`
    - Update relevant LLD documents
    - Update `tasks.md`
- Never implement without an LLD.
- If a task lacks an LLD reference → stop and ask for clarification.
- Maintain full traceability at all times.

## Workflow Chain
`Requirement` → `Plan` → `LLD` → `Task` → `Implementation`
